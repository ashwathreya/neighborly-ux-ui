## âœ… Feedback 6: Text-Heavy & Repetitive

### What to Fix:

**Current Issues:**
- Too much text on slides
- Repetitive content
- Documentation style vs. storytelling

**Solutions:**

1. **Visual-First Approach**
   - Large wireframes/mockups
   - Minimal text (6 words max per bullet)
   - Use icons and diagrams
   - Show, don't tell

2. **Storytelling Structure**
   - Problem â†’ Research â†’ Design â†’ Test â†’ Result
   - Each slide tells part of the story
   - Connect slides with narrative
   - Remove redundant information

3. **Slide Content Guidelines**
   - Title: 1 line
   - Body: 3-5 bullet points max
   - Each bullet: 6-8 words
   - Use visuals to explain

**Action Items:**
- [ ] Reduce text on all slides
- [ ] Replace text with visuals where possible
- [ ] Remove repetitive content
- [ ] Focus on storytelling
- [ ] Use speaker notes for details

## Presentation Tips

1. **Tell a Story**
   - Start with problem
   - Show research
   - Explain design process
   - Show testing
   - End with results

2. **Show, Don't Tell**
   - Large visuals
   - Minimal text
   - Let wireframes speak

3. **Use Data**
   - Every decision backed by research
   - Metrics for everything
   - User quotes for validation

4. **Be Honest**
   - Show challenges
   - Discuss trade-offs
   - Admit what didn't work

5. **Focus on UX Thinking**
   - Why, not just what
   - Process, not just outcome
   - User needs, not features
